    CSS namespacing

    Runtime package contains css files which could conflict with parent web site ( in case of embedding runtime app into web page ). Therefore SDK provides ability to resolve such sort of conflicts via generating namespaced css files. In case you have css conflicts you need to manually change path to css files inside styles.html.

    Example:

    <link href="{{staticsPath}}/assets/runtime-d9062fc459f15ab12eab.css" rel="stylesheet">

    to

    <link href="{{staticsPath}}/assets/namespacedCss/runtime-d9062fc459f15ab12eab.css" rel="stylesheet">

    Be aware that runtime root html element (which is configured via global.App.getRuntimeEntryPoint see Embedding runtime into page section) should have specific css class. By default it's __ but maybe changed at screen designer build time.
